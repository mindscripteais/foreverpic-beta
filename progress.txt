[2026-04-21] — OTTIMIZZAZIONE MEMORIA
Problema: Next.js 16.2.4 con Turbopack causava eccessivo RAM usage durante dev.

Azioni intraprese:
1. Semplificato next.config.js - rimosso configurazioni non valide
2. Rimossi .next/dev/types da tsconfig.json include (Next.js lo re-aggiunge automaticamente)
3. lib/auth.ts: caching tier in JWT token per evitare DB query ogni request
4. Corretto import duplicati in signin/page.tsx
5. Rimosso modularizeImports per lucide-react (causava build error)
6. Rinominato middleware.ts -> proxy.ts (deprecazione Next.js 16)
7. Build verificato: passa in 3.4s
8. Dev server avviato: risponde sulla porta 3000

OSSERVAZIONI:
- Next.js 16 usa Turbopack di default e NON supporta --no-turbopack per disabilitarlo
- Turbopack è la causa principale dell'elevato RAM usage
- Non c'è modo semplice di tornare a webpack in Next.js 16
- L'unica opzione sarebbe downgradare a Next.js 15

[2026-04-23] — FIX Create Event button
Problema: Il tasto "Create Event" nel modal non funzionava.

Analisi:
1. Il form aveva `required` sugli input che poteva bloccare il submit in modo invisibile
2. Il bottone aveva sia `type="submit"` che `onClick` che potevano interferire
3. Il middleware matcher non escludeva `/api`, causando potenziali problemi con tRPC

Fix applicati:
1. Aggiunto `noValidate` al form per disabilitare validazione HTML5 nativa
2. Rimosso `required` dagli input del modal
3. Rimosso `onClick` dal bottone "Create Event", lasciando solo il form `onSubmit`
4. Fixato middleware matcher per escludere `/api` dalle route protette
